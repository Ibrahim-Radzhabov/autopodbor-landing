Этот файл описывает проект «Автоподбор» для любого AI-агента, который будет работать с кодом. Прочитай его полностью перед тем, как вносить любые изменения.

## Что это за проект

Лендинг для сервиса автоподбора — профессиональной услуги по поиску, проверке и сопровождению покупки подержанного автомобиля. Целевая аудитория — покупатели б/у авто в России, которые боятся нарваться на скрытые дефекты или переплатить.

**Ключевые метрики, вшитые в копирайт:**
- 1 400+ проведённых осмотров
- 62% отбраковки (каждые 2 из 3 осмотренных машин отклоняются)
- Средний сторг — 120 000 ₽
- 5 звёзд, более 80 отзывов

## Секции лендинга (порядок в HTML)

1. **`<header>`** — Логотип + телефон + CTA кнопка. Sticky после скролла (класс `scrolled` добавляется JS).
2. **`<nav>` (mobile)** — Гамбургер-меню для мобильных. Открывается классом `open` на `<body>`.
3. **`#hero`** — Hero с H1, подзаголовком, 3 trust-бейджами, основной кнопкой CTA.
4. **`#process`** — Как мы работаем: 5 шагов (горизонтальный таймлайн на десктопе, вертикальный на мобиле).
5. **`#services`** — 3 тарифа (Консультация / Подбор под ключ / VIP). Featured-карточка — «Подбор под ключ».
6. **`#checklist`** — Что проверяем: сетка иконок + кнопка «Посмотреть пример отчёта».
7. **`#cases`** — Кейсы клиентов: горизонтальный scroll-snap карусель (5 карточек).
8. **`#trust`** — Доверие: фото эксперта, цитата, счётчики (animate on scroll).
9. **`#after-booking`** — Что происходит после заявки: 3 шага на светлом фоне.
10. **`#faq`** — FAQ-аккордеон: 6 вопросов, анимация max-height.
11. **`#final-cta`** — Финальный призыв + встроенная форма.
12. **`<footer>`** — Ссылка на privacy.html, копирайт, юридический текст.
13. **`#modalOverlay`** — Модальное окно формы (вызывается из hero и кнопок по сайту).
14. **`#reportOverlay`** — Модальное окно с примером отчёта (вызывается из #checklist).
15. **`.sticky-cta`** — Липкая панель снизу на мобиле (появляется после прокрутки hero).
16. **`.floating-widget`** — Плавающая кнопка звонка (скрывается при видимости sticky-cta).

## Конфигурация (JS CONFIG)

Весь конфиг — в одном объекте в начале `<script>` блока:

```javascript
var CONFIG = {
  WEB3FORMS_KEY: 'YOUR_WEB3FORMS_ACCESS_KEY',  // https://web3forms.com
  TELEGRAM_BOT_TOKEN: 'YOUR_BOT_TOKEN',         // @BotFather → /newbot
  TELEGRAM_CHAT_ID: 'YOUR_CHAT_ID',             // ID чата куда приходят заявки
  METRIKA_ID: 'XXXXXXXX',                       // Номер счётчика без кавычек в ym()
  PHONE: '+79001234567'                         // Номер телефона (используется в ссылках)
};
```

**Важно:** `METRIKA_ID` используется в двух местах — в теге `<script>` Метрики в `<head>` (там хардкод `XXXXXXXX`) и в `CONFIG.METRIKA_ID`. При замене нужно поменять оба места.

## Форма и обработка заявок

### Валидация
- Имя: непустое, после trim
- Телефон: 11 цифр, начинается с 7 (формат `+7 (___) ___-__-__`)
- Чекбокс согласия: обязателен
- Ошибки показываются под каждым полем через `.field-error`

### Отправка
Параллельно (Promise.all) в два канала:
1. **Telegram Bot API** — `https://api.telegram.org/bot{TOKEN}/sendMessage` — мгновенное уведомление
2. **Web3Forms** — `https://api.web3forms.com/submit` — email-доставка и архивирование

Если токены не заменены (равны заглушкам), функции тихо пропускают отправку — это для локальной разработки.

### UTM-параметры
Автоматически читаются из URL при загрузке и вставляются в скрытые поля формы. Передаются в Telegram и Web3Forms.

### Цели Метрики
```javascript
trackGoal('lead_form_open')    // открытие модального окна
trackGoal('lead_submitted')    // успешная отправка формы
trackGoal('phone_click')       // клик по телефону
trackGoal('whatsapp_click')    // клик по WhatsApp
```

## Анимации и интерактивность

- **IntersectionObserver** — запускает CSS-анимации при появлении в вьюпорте (класс `.visible` на `.animate-on-scroll`)
- **Счётчики** (`#trust` секция) — `requestAnimationFrame`, ease-out, запускаются один раз при появлении
- **Sticky header** — класс `scrolled` добавляется при `scrollY > 10`
- **Sticky CTA bar** — показывается после ухода hero за вьюпорт (только mobile)
- **Floating widget** — скрывается когда sticky-cta видна
- **FAQ аккордеон** — `max-height` + `opacity` transition, chevron поворот 180°
- **Phone mask** — `+7 (___) ___-__-__`, форматирование на каждый `input`-event
- **Grain overlay** — SVG `feTurbulence` в CSS `background-image`

### prefers-reduced-motion
Все анимации отключаются при `@media (prefers-reduced-motion: reduce)` — элементы сразу показываются в финальном состоянии.

## Мобильная адаптация

- **Breakpoints:** 767px (mobile), 768px (tablet+), 1024px (desktop)
- **Service cards:** на мобиле featured-карточка поднимается вверх через `order: -1`
- **Cases carousel:** `scroll-snap-type: x mandatory`, горизонтальный скролл
- **Modal:** bottom sheet на мобиле (`bottom: 0; border-radius: 24px 24px 0 0`), центрированное окно на десктопе
- **Process timeline:** вертикальный на мобиле, горизонтальный на десктопе

## Конвенции кода

1. **JS синтаксис** — `var`, не `const/let`. Функции через `function name(){}`, не стрелочные. Причина: максимальная совместимость, проект не билдится.
2. **CSS порядок** — сначала `:root` токены, потом reset, потом компоненты в порядке HTML-структуры.
3. **Комментарии в HTML** — `<!-- TODO: ... -->` для всех заглушек, требующих замены.
4. **Комментарии в JS** — `// TODO: ...` и `// ========== SECTION ==========` разделители.
5. **Нет инлайн-стилей** — только CSS-классы и custom properties.
6. **Никаких `!important`** — только если это абсолютно неизбежно для override внешнего стиля.
